# Security Policy

## Reporting Vulnerabilities

- Report vulnerabilities to security@pesglobalgroup.com
- Do not open public issues for sensitive findings
- We will respond within 48 hours

## Security Best Practices

- Use least privilege when creating PATs, service principals, and deploy keys
- Never commit secrets, API keys, or credentials to repositories
- Enable 2FA for all accounts
- Review and rotate secrets regularly

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | Limited   |
